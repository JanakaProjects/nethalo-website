### 1. Install backend dependencies
Run: npm install express better-sqlite3 bcryptjs jsonwebtoken cors tsx
Then: npm install -D @types/express @types/better-sqlite3 @types/bcryptjs @types/jsonwebtoken @types/cors

### 2. Create server/ directory with these files:

#### server/index.ts
Express server on port 3001 that:
- Serves static files from ../dist (in production mode)
- Has CORS enabled for localhost:5173 (dev mode)
- Has JSON body parser
- Mounts all route files under /api
- Auto-creates DB tables and seeds demo data on startup
- Logs a startup message on connect

#### server/db.ts
SQLite database setup using better-sqlite3:
- Store DB at ./nethalo.db
- On init, create these tables (IF NOT EXISTS):

1. users:
   - id TEXT PRIMARY KEY
   - name TEXT NOT NULL
   - email TEXT UNIQUE NOT NULL
   - password TEXT NOT NULL (bcrypt hashed)
   - role TEXT NOT NULL CHECK(role IN ('student','parent','admin'))
   - school TEXT
   - age INTEGER
   - child_name TEXT (for parent accounts)
   - admin_role TEXT (for admin accounts)
   - created_at TEXT DEFAULT CURRENT_TIMESTAMP

2. connected_accounts:
   - id INTEGER PRIMARY KEY AUTOINCREMENT
   - user_id TEXT NOT NULL REFERENCES users(id)
   - platform TEXT NOT NULL
   - label TEXT NOT NULL
   - connected INTEGER DEFAULT 0
   - username TEXT
   - connected_at TEXT
   - UNIQUE(user_id, platform)

3. threats:
   - id INTEGER PRIMARY KEY AUTOINCREMENT
   - user_id TEXT NOT NULL REFERENCES users(id)
   - platform TEXT NOT NULL
   - type TEXT NOT NULL
   - severity TEXT NOT NULL CHECK(severity IN ('low','medium','high','critical'))
   - message TEXT NOT NULL
   - timestamp TEXT DEFAULT CURRENT_TIMESTAMP
   - blocked INTEGER DEFAULT 1

4. reports:
   - id INTEGER PRIMARY KEY AUTOINCREMENT
   - user_id TEXT NOT NULL REFERENCES users(id)
   - type TEXT NOT NULL
   - description TEXT
   - target_name TEXT
   - created_at TEXT DEFAULT CURRENT_TIMESTAMP
   - resolved INTEGER DEFAULT 0

5. journal_entries:
   - id INTEGER PRIMARY KEY AUTOINCREMENT
   - user_id TEXT NOT NULL REFERENCES users(id)
   - content TEXT NOT NULL
   - mood TEXT
   - created_at TEXT DEFAULT CURRENT_TIMESTAMP

6. children:
   - id INTEGER PRIMARY KEY AUTOINCREMENT
   - parent_id TEXT NOT NULL REFERENCES users(id)
   - name TEXT NOT NULL
   - grade TEXT
   - platform TEXT DEFAULT 'Instagram'
   - status TEXT DEFAULT 'online'
   - threats_detected INTEGER DEFAULT 0
   - last_active TEXT
   - safe_score INTEGER DEFAULT 90

### 3. server/routes/auth.ts
- POST /api/auth/signup — creates user with bcrypt-hashed password, returns JWT + user object (without password)
- POST /api/auth/login — validates email+password, returns JWT + user object (without password)
- GET /api/auth/me — requires JWT, returns current user data
- PUT /api/auth/me — requires JWT, updates name/email/school/age
- The JWT should contain: { userId: user.id, role: user.role } and expire in 7 days
- JWT secret from process.env.JWT_SECRET or fallback to 'nethalo-dev-secret-key-change-in-production'
- On signup, also seed the 6 default connected_accounts for the new user (all connected=0)

### 4. server/routes/dashboard.ts
All require JWT auth middleware:

GET /api/dashboard/stats?role=student
- Returns student dashboard stats: threat count, active alerts, recent events, reports, journals

GET /api/dashboard/stats?role=parent
- Returns parent dashboard: children list with status/safe scores, active alerts, reports this month, recent updates

GET /api/dashboard/stats?role=admin
- Returns admin dashboard: total students, active/reports/resolved counts, safety score, student list, pending reports, recent actions

### 5. server/routes/threats.ts
